### 2. ✅ **Dark e Light Mode**
- **Arquivo:** `frontend/src/contexts/ThemeContext.jsx`
- **Características:**
  - Context React para gerenciar tema globalmente
  - Detecta preferência do sistema operacional
  - Salva preferência no localStorage
  - Integrado com Tailwind CSS (classe `dark:`)
  - Toggle botão na sidebar

**Como usar:**
```javascript
import { useTheme } from '@/contexts/ThemeContext';

function MyComponent() {
  const { isDark, toggleTheme } = useTheme();
  
  return <button onClick={toggleTheme}>
    {isDark ? '☀️ Claro' : '🌙 Escuro'}
  </button>;
}
```

### 5. ✅ **Categorias Salvas no localStorage**
- **Arquivo:** `frontend/src/hooks/useCategories.js`
- **Características:**
  - Hook personalizado para gerenciar categorias
  - Carrega categorias padrão na primeira vez
  - Salva categoria novas automaticamente
  - Suporta adicionar/remover categorias customizadas
  - Evita duplicatas

**Como usar:**
```javascript
import { useCategories } from '@/hooks/useCategories';

function TransactionForm() {
  const { categories, addCategory } = useCategories();
  
  // categories.income — lista de categorias de renda
  // categories.expense — lista de categorias de despesa
  
  // Adicionar categoria customizada
  addCategory('income', 'meu_freelance');
}
```

### 6. ✅ **Infinite Scroll para Transações**
- **Arquivo:** `frontend/src/hooks/useInfiniteScroll.js`
- **Características:**
  - Hook que usa IntersectionObserver API
  - Sensível ao scroll da página (threshold 10%)
  - Trigger ao final do feed de transações
  - Configurable (rootMargin, threshold)
  - Performance otimizado (não causa lag)

**Como usar:**
```javascript
import { useInfiniteScroll } from '@/hooks/useInfiniteScroll';

function TransactionList() {
  const sentryRef = useInfiniteScroll(() => {
    // Carregar mais transações quando atinge fim
    loadMoreTransactions();
  });

  return (
    <>
      {transactions.map(tx => <TransactionItem key={tx.id} {...tx} />)}
      <div ref={sentryRef} className="h-4" /> {/* Sentry element */}
    </>
  );
}
```

### 7. ✅ **Confirmação de E-mail no Registro**
- **Arquivo:** `backend/src/services/emailService.js`
- **Características:**
  - Envia e-mail de confirmação após registro
  - Template HTML profissional
  - Token único com expiração de 24 horas
  - Link de confirmação personalizado
  - Integrado com Nodemailer

**Fluxo:**
1. Usuário se registra
2. E-mail de confirmação enviado
3. Token salvo no banco (com timestamp)
4. Usuário clica no link
5. E-mail confirmado, conta ativa

**Configuração .env necessária:**
```
EMAIL_SERVICE=gmail  # ou outro serviço
EMAIL_USER=seu-email@gmail.com
EMAIL_PASSWORD=sua-senha-app
EMAIL_FROM=noreply@financetrack.com
FRONTEND_URL=https://seu-site.com
```

### 8. ✅ **Recuperação de Senha**
- **Arquivo:** `backend/src/services/emailService.js`
- **Características:**
  - Endpoint para solicitar recuperação
  - E-mail com link de reset (expira em 1 hora)
  - Token criptografado
  - Template HTML com instruções
  - Proteção contra force brute

**Fluxo:**
1. Usuário clica "Esqueci minha senha"
2. Coloca e-mail
3. E-mail de recuperação enviado
4. Link com token válido por 1 hora
5. Usuário define nova senha
6. Senha resetada

**Endpoints necessários:**
```
POST /api/v1/auth/forgot-password
POST /api/v1/auth/reset-password
GET /api/v1/auth/verify-reset-token
```

### 9. ✅ **Módulo de Investimentos**
- **Arquivo:** `backend/src/routes/investments.js`
- **Características:**
  - CRUD completo de investimentos
  - Tipos: stock, crypto, bond, real_state, fund, other
  - Rastreamento de ganhos (current - initial)
  - Cálculo automático de percentual de ganho
  - Resumo consolidado de investimentos

**Endpoints:**
```
GET    /api/v1/investments           — Listar meus investimentos
POST   /api/v1/investments           — Criar investimento
PUT    /api/v1/investments/:id       — Atualizar investimento  
DELETE /api/v1/investments/:id       — Deletar investimento
GET    /api/v1/investments/summary   — Resumo total de ganhos
```

**Exemplo de criação:**
```json
{
  "name": "Tesla Stock",
  "type": "stock",
  "initial_amount": 10000,
  "current_value": 12500
}
```

**Resposta com ganhos calculados:**
```json
{
  "id": 1,
  "name": "Tesla Stock",
  "type": "stock",
  "initial_amount": 10000,
  "current_value": 12500,
  "gains": 2500,
  "gains_percent": 25.00,
  "created_at": "2026-04-14T12:00:00Z"
}
```

## 🗄️ Banco de Dados - Tabela Investments

```sql
CREATE TABLE IF NOT EXISTS investments (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  type TEXT NOT NULL CHECK(type IN ('stock', 'crypto', 'bond', 'real_state', 'fund', 'other')),
  initial_amount NUMERIC NOT NULL,
  current_value NUMERIC NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
